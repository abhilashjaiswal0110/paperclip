Facilitate the weekly retrospective across all teams and capture learnings for continuous improvement.

## Checklist

1. Collect written retrospective input from CTO, CMO, and UX Designer before the session
2. Summarise: what went well, what didn't, what to improve
3. Review last week's improvement actions — were they completed?
4. Define 1–3 concrete improvement actions for next week with named owners
5. Review sprint progress against the product-mvp roadmap milestones
6. Update ROADMAP.md if any timelines or scope have changed
7. Share retrospective summary with all team leads
8. Log any strategic decisions made into the CEO memory files
