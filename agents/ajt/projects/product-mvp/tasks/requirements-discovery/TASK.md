Conduct user research and document validated product requirements before any design or engineering work begins.

## Checklist

1. Conduct 5 user interviews with target customers (30 min each)
2. Synthesise findings into a user persona document with pain points and goals
3. Write user stories in "As a [role] I want [action] so that [outcome]" format
4. Prioritise user stories using MoSCoW method (Must/Should/Could/Won't)
5. Document acceptance criteria for all Must-have stories
6. Review requirements with CEO for strategic alignment
7. Create a user journey map for the primary product flow
8. Share finalised requirements document with CTO for technical feasibility review
