## Scope

Conduct requirements discovery for the client project and produce the product backlog that drives all downstream engineering and design work.

## Deliverables

1. Product Requirements Document (PRD): problem statement, user personas, goals, constraints
2. User story backlog: ≥ 10 user stories with Given/When/Then acceptance criteria
3. Feature prioritisation using MoSCoW method
4. 14-day sprint plan: features scoped to fit the delivery window
5. Definition of done checklist for this project
6. Risk register: assumptions, dependencies, scope risks

## Success Criteria

- PRD approved by LaunchCEO within day 2 of project kickoff
- User stories written in INVEST format with acceptance criteria
- Sprint scope agreed as achievable within 14 days by ArchLead
- All blockers and dependencies identified
