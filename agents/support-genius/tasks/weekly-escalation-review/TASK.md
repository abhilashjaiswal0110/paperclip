---
name: Weekly L2/L3 Escalation Review
assignee: support-director
recurring: true
---

Review all L1-to-L2 and L2-to-L3 escalations from the prior week to identify patterns, close the training loop, and improve deflection rates.

## Agenda

1. Pull all tickets escalated from L1 to L2 in the past 7 days
2. Classify escalation reasons: knowledge gap, permissions required, complex investigation, VIP customer
3. For knowledge-gap escalations: create a BotCoach task to close the gap in the KB
4. Pull all tickets escalated from L2 to L3 in the past 7 days
5. Classify L3 escalation reasons: engineering defect, API/vendor failure, security incident, novel issue
6. For any escalation taking more than 2x the SLA target, identify root cause and create a process improvement task
7. Update L1 and L2 deflection rate metrics for the week
8. Post escalation review summary as task comment: total escalations, top 3 reasons, deflection rate trend, action items created

## Output

- Weekly escalation report posted as task comment
- Action items created in Paperclip for BotCoach (knowledge gaps) and SLAGuard (SLA breaches)
- SupportDirector @-mentions SupportCEO if escalation rate is trending up for 3 or more consecutive weeks
